How to learn Rails - guides and tutorials.

A friend learned some Ruby on Rails a few years ago (Rails 1.1). Now he's returning to web development, and he wants to get back up to speed fast. He asked me if I had any advice. Turns out I did.

1. The Rails Tutorial. http://ruby.railstutorial.org/ruby-on-rails-tutorial-book This is a free, online, complete book-length tutorial for learning rails. It's pretty comprehensive for learning rails, and it's great.

It starts in with testing early, in the third chapter. Obey the paradigm; do TDD (test-driven development)! A huge strength of the book is that it integrates testing from the beginning, rather than relegating it to a late-chapter afterthought. If a book presents testing in its own chapter, then it's not going to be effective in teaching the TDD philosophy.

RSpec is nearly universally-used in the workplace, in my experience.

2. You are using git, right? Good source control is powerful and pretty liberating. Git is the standard, and of course there's Github, which is ridiculously useful for using, publishing, and collaborating on open-source code.

You might want to install an up-to-date version of git and download the HTML docs for that version at the same time. As git gets updated, sometimes the online kernel.org documentation changes without notice. (there's no note of what version each page refers to.) I've been bitten before by relying on the online docs, which had been updated to reflect new functionality that my version of git didn't have.

git high-level doc: http://www.kernel.org/pub/software/scm/git/docs/everyday.html -- "Everyday GIT With 20 Commands Or So"

git books: http://gitready.com http://progit.org

git downloads: http://www.kernel.org/pub/software/scm/git/git-1.7.6.tar.bz2 http://www.kernel.org/pub/software/scm/git/git-htmldocs-1.7.6.tar.bz2

3. Install and use rvm as soon as possible. I hated it & didn't get it at first. But eventually I realized rvm's power. It is fabulous. The sooner you start using it, the sooner you will benefit, especially as you are on a learning curve and will probably be installing lots of gems. rvm sandboxes your projects and makes everything MUCH easier to keep clean.

For example, if you upgrade ruby or rails for one project, or install or upgrade any gem (RSpec, CanCan, etc.), you might start seeing things fail on another project. And it can get very painful to isolate which change (if any) is causing the problem. With rvm, Rails 3's Gemfiles, and the bundler gem, you can isolate almost all of that per project: ruby version, bundler version, rails & gem versions, etc.

rvm: https://rvm.beginrescueend.com/ Make sure you follow the complete install instructions, including the .profile specifics and the 'default ruby' detail. After that, the rvm docs on the site are thorough, but irritatingly organized. Oh well. You can ask me rvm questions.

z) Finally, a bit about Behavior-Driven Development, and my favorite article on TDD and BDD.

BDD (behaviour-driven development), usually using Cucumber, is pretty popular and widely-discussed. But where it's used, it's almost always in concert with using Rspec anyway. Doing TDD with Rspec is considered best practice almost everywhere, and you should start there.

That being said, you might want to read this blog post on "Outside-In Development" (BDD with Cucumber and Rspec). It easily demonstrates how the philosophies and tools provide different benefits. Also, there's a bit about where not to use TDD/Rspec. http://www.sarahmei.com/blog/2010/05/29/outside-in-bdd/
